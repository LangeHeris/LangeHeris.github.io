服务端结构

主要结构

DB_XMLServer
Authenticator
LoginServer
Coordinator
GameServer
NpcClient

服务端启动顺序及其作用

  1. DB_XMLServer\XMLDB.exe
  2. Authenticator\Authenticator_Console.exe
  3. LoginServer\LoginServer.exe setting:”file://data/features.xml=”[]”
  4. Coordinator\Coordinator.exe setting:”file://data/features.xml=”[]”
  5. GameServer\GameServer.exe setting:”file://data/features.xml
  6. NPCClient\ClientD.exe [IP 端口] setting:”file://data/features.xml=[] [batchjob]

3和4 调用服务器频道的数据，channelindexinfo。

5和6 调用”系统提示”的数据，分别为gamesever skill standard 三个部分的系统提示信息。
还有NPC对话的文本，NPC名，地名，怪物编号和对话文本，道具名等相关代码

数据存储

DB_XMLServer 存放玩家数据

例如账号account，角色character，宠物pet，农场family，公会guild，房子house 等等。

character文件夹下存放角色数据 ，可编辑角色的头衔、技能、关键字、道具、任务、三维、等级……

修改后重新登录服务器即可生效。

Authenticator 创建角色卡

DataGenerator.exe 生成角色或宠物卡片

datagenerator.config.xml 储存DataGenerator.exe的文本信息

LoginServer

生成服务器主机的信息日志

游戏登陆运行环境，例如频道名称、服务器IP地址、数据库端口等等.

ServerInfo.ini 可以设置

The name of the Server.

The Login port for the server (Must be open)

NPC Client Port

Coordinator Port

XMLDB Database

XMLDB Database Port

Coordinator

协调器，协调服务器端口、角色创建程序。

检测登陆程序IP和端口

检测数据库地址和端口

GameSever

存放游戏系统、技能数据的库

NPCClient

专门给游戏服务器刷怪的NPC账号所使用的客户端。

踩坑记录

利用代码获得的关键字和任务会导致整条主线无法正常进行，目前只能通过正常游戏流程攻略。
建议多备份角色数据，或者直接修改角色数据。

任务发放时间看的是电脑时间。（也就是主机时间）

部分利用代码获得的任务道具，NPC无法识别。

已知修改游戏任务代码也无法让不能正常进行的任务强制执行。（我当时改的可能是代码副本，游戏没有调用23333）
